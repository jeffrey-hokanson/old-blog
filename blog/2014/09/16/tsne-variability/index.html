Are tSNE/viSNE Embeddings Reproducible?

On September 10, 2014 I gave a talk to the MD Anderson Flow Cytometry User Group. One persistent question was "are viSNE runs reproducible?" It depends on what you mean by reproducible. If you expect broadly similar patterns in the 'eyeball norm', then yes; but if you expect identitical patterns, unfortunately, they are not. To demonstrate, I've run the t-SNE algorithm multiple times on the same data and compared the results. In this example, we use clinical data from an AML patient's peripheral blood 5 hours after treatment.

The embedding that t-SNE uses is invariant under translation, rotation, and reflection; as such, we cannot expect each resulting plot to be identitical. (Hence the importance to compare t-SNE embeddings that were computed at the same time!) However, given two of these plots we can find the rotation and reflection of one to best match another by solving the Orthgonal Procrustes Problem. After this modification, the plots look similar.

Looking at All the Data

By default in both SightOf (aka viSNE) and flowML take a random sample of points to reduce computational complexity; this introduces an additional degree of randomness. However, even with the same points, if we compute the t-SNE embedding starting from a different random initial guess, the solution will be different. This is because there are many local minima which the algorithm could converge to. Finding the global minima is for nonconvex problems like t-SNE is an unsolved problem in mathematics and one for which we do not expect a solution.

In addition to running additional iterations, another approach to get a more 'definiative' plot would be to run multiple iterations and simply select the one with the minimum residual. However, this would be expensive because each run of the Barnes-Hut t-SNE on full datasets takes about 30 minutes. The moral of the story is simple: although the general patterns of a particular t-SNE embedding appear in other solutions, no two embeddings are identical. None the less, the t-SNE embedding is still an appropreate tool for visualization and gating.
